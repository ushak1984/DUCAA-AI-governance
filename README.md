# DUCAA AI Governance Policy

This repository presents a short academic portfolio project on the **Dual-Use Cybersecurity AI Accountability Act (DUCAA)**.

DUCAA is a proposed policy framework for responsible AI governance in cybersecurity. It focuses on how defensive AI tools can become risky when attackers misuse trusted systems, third-party integrations, OAuth access, and social engineering.

## Project Files

- **DUCAA-AI governance.pdf**  
  Full policy proposal for ethical AI governance in cybersecurity.

- **DUCAA-AI governance Case study.pdf**  
  Case study on the 2025 Workday social engineering breach.

## Project Summary

The project explains why AI-based cybersecurity systems need stronger rules, audits, and accountability. The Workday case shows that attackers can bypass AI defenses by making harmful actions look like normal user activity.

## Key Ideas

- Dual-use AI risk
- Cybersecurity ethics
- Social engineering and OAuth abuse
- Vendor accountability
- AI audit and documentation
- Multi-factor authentication
- Supply-chain security

## DUCAA Policy Recommendations

DUCAA recommends:

1. Transparent AI documentation and logging
2. Pre-deployment ethical audits
3. Restricted access to sensitive AI systems
4. Regular dual-use risk assessments
5. Strong vendor and supply-chain controls
6. Mandatory multi-factor authentication for OAuth, API, and admin access
